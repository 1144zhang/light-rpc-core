# light-rpc-core 从零到一学习路线

## 一、项目定位

`light-rpc-core` 是一款从零实现的轻量级 RPC 框架，核心目标是帮助你：

1. **深入理解 RPC 原理**：不再是 Dubbo/gRPC 的黑盒使用者，而是理解每一层的工作机制
2. **掌握分布式基础知识**：注册中心、负载均衡、序列化、网络通信、动态代理
3. **应对面试深挖**：每个模块都能讲清楚「为什么这样设计」「踩了什么坑」「怎么优化的」
4. **打造简历亮点**：自研轮子项目远优于 CRUD 管理系统

---

## 二、前置知识要求

开始学习前，建议掌握以下基础：

| 知识点 | 最低要求 | 建议学习资源 |
|--------|----------|------------|
| Java 基础 | 集合、泛型、注解、反射 | 《Java 核心技术》 |
| Java 并发 | 线程池、synchronized、volatile、CompletableFuture | 《Java 并发编程的艺术》 |
| 动态代理 | 理解 JDK Proxy 基本原理 | 能说出 Proxy.newProxyInstance 三个参数 |
| 网络基础 | TCP 三次握手/四次挥手、OSI 七层模型 | 《TCP/IP 详解》卷一前 5 章 |
| Maven | 分模块构建、依赖管理 | 会用 mvn clean install |
| Zookeeper | 基础概念（节点、Watch、临时节点） | ZK 官方文档前 3 章 |

如果你某块基础薄弱，建议先花 **1-2 周**补基础，再进入项目学习。

## 三、学习阶段（共 6 周）

### 第 1 周：RPC 理论基础 + 项目跑通

**目标**：理解 RPC 是什么、为什么用、核心流程是怎样的

```
Day 1-2: 阅读「RPC 基础理论」文档，理解 RPC 核心概念和调用链路
Day 3-4: 阅读「项目架构设计」文档，理解整体架构和模块职责
Day 5-6: 把项目跑起来（启动 ZK → 启动 Server → 启动 Client）
Day 7:   跟一遍完整调用链路（Client 发请求 → Server 处理 → 返回结果）
```

**检验标准**：能把 RPC 调用链路用自己的话讲清楚

---

### 第 2 周：网络传输层（Netty + 自定义协议）

**目标**：理解 RPC 框架的「骨架」——数据怎么在网络上传输

```
Day 1-2: 学习 Netty 基础（BIO→NIO→Reactor 模型演进）
Day 3-4: 阅读「网络传输层详解」文档，精读以下核心类：
         - RpcMessageCodec / RpcMessageEncoder / RpcMessageDecoder
         - NettyRpcServer / NettyRpcClient
         - ChannelProvider（连接复用）
Day 5:   理解自定义协议设计（魔数 + 序列化器编号 + 长度 + 体）
Day 6:   研究粘包/拆包解决方案（LengthFieldBasedFrameDecoder）
Day 7:   写笔记总结：ByteToMessageDecoder 和 MessageToByteEncoder 的区别
```

**核心类清单**：
- `remoting/transport/netty/codec/RpcMessageDecoder.java`
- `remoting/transport/netty/codec/RpcMessageEncoder.java`
- `remoting/transport/netty/codec/RpcMessageFrameDecoder.java`
- `remoting/transport/netty/server/NettyRpcServer.java`
- `remoting/transport/netty/client/NettyRpcClient.java`
- `remoting/transport/netty/client/ChannelProvider.java`

**检验标准**：能画出 Netty 的 ChannelPipeline 处理流程图

---

### 第 3 周：序列化机制（Kryo + Protostuff + Hessian）

**目标**：理解数据怎么从对象变成字节流，以及不同序列化方式的权衡

```
Day 1-2: 学习序列化基础概念（什么是序列化、常见方式对比）
Day 3-4: 阅读「序列化机制详解」文档，精读：
         - Serializer 接口设计
         - KryoSerializer（重点：ThreadLocal 保证线程安全）
         - ProtostuffSerializer / HessianSerializer
Day 5:   理解 SPI 机制如何在序列化中应用
Day 6:   跑 JMH 基准测试（如果没有就手动对比耗时和体积）
Day 7:   写对比表格：JSON vs JDK vs Kryo vs Protostuff vs Hessian
```

**核心类清单**：
- `serialize/Serializer.java`
- `serialize/kyro/KryoSerializer.java`
- `serialize/protostuff/ProtostuffSerializer.java`
- `serialize/hessian/HessianSerializer.java`

**检验标准**：能说清楚「为什么 Kryo 比 JDK 序列化快」「Kryo 线程安全问题怎么解决」

---

### 第 4 周：注册中心 + 动态代理 + 负载均衡

**目标**：理解 RPC 框架的「大脑」——服务怎么被发现和调用

```
Day 1-2: 阅读「注册中心详解」文档，精读 ZK 相关代码
Day 3-4: 阅读后续文档（动态代理 + 负载均衡），精读：
         - RpcClientProxy（JDK 动态代理核心）
         - CuratorUtils（ZK 操作工具类）
         - ConsistentHashLoadBalance（一致性哈希实现）
Day 5:   研究 CompletableFuture 如何实现异步调用
Day 6:   研究服务分组 group 和版本号 version 的实现
Day 7:   画图总结：从 Client 发起调用到 Server 返回结果的完整数据流
```

**核心类清单**：
- `proxy/RpcClientProxy.java`
- `registry/zk/ZkServiceDiscoveryImpl.java`
- `registry/zk/ZkServiceRegistryImpl.java`
- `loadbalance/loadbalancer/ConsistentHashLoadBalance.java`
- `loadbalance/loadbalancer/RandomLoadBalance.java`

**检验标准**：能说出「从调用接口方法到 Server 执行，中间经历了哪些步骤」

---

### 第 5 周：Spring 集成 + 辅助功能

**目标**：理解框架如何优雅地集成到业务系统

```
Day 1-2: 研究注解驱动机制（@RpcService / @RpcReference / @RpcScan）
Day 3-4: 研究 Spring 集成（CustomScanner、SpringBeanPostProcessor）
Day 5:   研究心跳检测（IdleStateHandler 配置和使用）
Day 6:   研究 Gzip 压缩（Compress 接口和实现）
Day 7:   研究 SPI 扩展加载器（ExtensionLoader）
```

**核心类清单**：
- `spring/CustomScanner.java`
- `spring/SpringBeanPostProcessor.java`
- `compress/gzip/GzipCompress.java`
- `extension/ExtensionLoader.java`

## 六、核心原则

> **「不要只看代码，要理解设计决策」**

每个模块学习时，问自己三个问题：
1. **为什么这样设计？**（有没有更好的方案？）
2. **遇到了什么问题？**（哪个坑让你印象深刻？）
3. **如果流量涨 10 倍，哪里先崩溃？**（怎么扩容？）

这三个问题的答案，就是面试时最有力的武器。
